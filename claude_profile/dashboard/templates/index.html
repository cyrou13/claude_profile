Guide d'utilisation

Installation rapide
# Cloner et installer
git clone git@github.com:cyrou13/claude_profile.git
cd claude_profile

# Option 1 : script tout-en-un (installe + lance le dashboard)
./start.sh

# Option 2 : install manuelle
uv sync --dev

# Rendre la commande disponible globalement
uv tool install -e .

# Ou executer via uv run (sans install globale)
uv run claude-profile --help
Sync multi-machines

Synchronise tes skills, agents, commands et CLAUDE.md entre Mac et Linux via un repo Git prive.

# Premier setup (cree le repo de sync)
uv run claude-profile sync init

# Voir ce qui a change
uv run claude-profile sync status

# Envoyer les modifs locales
uv run claude-profile sync push

# Recuperer les modifs de l'autre machine
uv run claude-profile sync pull
Ce qui est synchronise
  • CLAUDE.md — instructions globales
  • settings.json — configuration Claude Code
  • skills/ — tous les fichiers SKILL.md + scripts
  • agents/ — definitions d'agents
  • commands/ — commandes custom
  • CLAUDE.md par projet — stockes par nom normalise
Ce qui est exclu

Sessions, historique, cache, telemetrie, stats, plans, IDE, fichiers temporaires.

Tip : Si tu as installe globalement avec uv tool install -e ., tu peux utiliser claude-profile directement sans le prefixe uv run.

Profils perso / pro

Isole les contextes entre projets personnels et professionnels via des overlays CLAUDE.md.

# Lister les profils
uv run claude-profile profile list

# Activer un profil (modifie ~/.claude/CLAUDE.md)
uv run claude-profile profile activate avicenna

# Voir le profil actif et ses projets
uv run claude-profile profile show

# Creer un nouveau profil
uv run claude-profile profile create devops \
    --projects "infra,monitoring,ci-cd" \
    --description "Projets DevOps"
Comment ca marche ?

L'activation d'un profil ajoute un overlay en bas de ton CLAUDE.md global. Cet overlay donne du contexte a Claude Code sur le perimetre actif. Le dashboard filtre aussi les sessions et stats par profil.

Dashboard

Interface web pour visualiser ton usage de Claude Code.

# Lancer le dashboard (port 8741 par defaut)
uv run claude-profile dashboard

# Port custom
uv run claude-profile dashboard --port 9000

# Ou via le script (gere tout automatiquement)
./start.sh
CLAUDE_PROFILE_PORT=9000 ./start.sh
OngletContenu
UsageMessages, sessions, tokens, activite quotidienne, repartition par modele
SessionsTableau detaille filtrable par profil — duree, tokens, outcome
ProfilsStats agregees par profil (sessions, tokens, fichiers)
VeilleReleases Claude Code, articles dev.to + HN, repos communautaires
RecommandationsSuggestions d'optimisation : choix de modele, duree sessions, taux d'echec
Veille en CLI

Consulter les dernieres nouveautes sans ouvrir le dashboard.

uv run claude-profile veille

Sources : releases GitHub (anthropics/claude-code, modelcontextprotocol/servers), articles dev.to, Hacker News, repos communautaires.

Configuration

Fichier : ~/.config/claude-profile/config.toml

Cree automatiquement par ./start.sh au premier lancement si absent.

[general]
claude_home = "~/.claude"
sync_repo = "~/Documents/dev/claude-profile-sync"

[sync]
include = ["CLAUDE.md", "settings.json", "skills/", "agents/", "commands/"]
scan_dirs = ["~/Documents/dev"]

[veille]
check_interval_hours = 24
github_repos = [
    "anthropics/claude-code",
    "modelcontextprotocol/servers",
]

[dashboard]
port = 8741

[[profiles]]
name = "avicenna"
projects = ["cortex", "aria", "reportia"]
claude_md_overlay = """
## Contexte : Avicenna.ai
- Focus medical device SaaS
"""

Setup sur une seconde machine
# 1. Cloner le projet
git clone git@github.com:cyrou13/claude_profile.git
cd claude_profile

# 2. Installer
uv sync --dev

# 3. Copier ta config (ou la recreer)
# Le fichier est dans ~/.config/claude-profile/config.toml

# 4. Recuperer la config Claude Code synchronisee
uv run claude-profile sync pull

# 5. Lancer le dashboard
./start.sh
